This reference condenses the official COMSOL 6.4 documentation most relevant to AI-driven operation of COMSOL on a local machine.

## What the API Can Control

According to the Programming Reference intro, the COMSOL API is usable from:

- the Application Builder;
- model methods;
- the Shell window in the COMSOL Desktop;
- model files for Java;
- LiveLink for MATLAB.

Use these paths when the user asks for deterministic scripted model construction or repeatable edits instead of manual GUI steps.

## Safe Model Loading and Saving

The `ModelUtil` API exposes:

- `create(...)` and `createUnique(...)` for new models;
- `load(...)` for loading an existing model file;
- `loadCopy(...)` for loading a copy that must later be saved to a new filename.

Prefer `loadCopy(...)` when modifying an existing `.mph` so the original source model remains untouched until the workflow succeeds.

## Batch and Method Workflows

The local 6.4 launcher help confirms these important `comsolbatch` options:

- `-inputfile` and `-outputfile`
- `-study`
- `-batchlog`
- `-methodcall`
- `-methodinputfile`
- `-paramfile`
- `-pname`, `-plist`, and `-pindex`
- `-usebatchlic`

Use batch mode when the model structure already exists and the goal is solve/re-solve/export rather than interactive editing.

## Server / Client

The local `comsolmphserver -help` output confirms that COMSOL Multiphysics Server supports:

- `-port`
- `-portfile`
- `-multi`
- `-graphics`
- login control options

Use server mode when the workflow depends on client/server separation, repeated client connections, or MATLAB integration through the server.

## Exports and Reports

The Introduction manual documents that result plot groups can add an Image export node, and images can be exported from the Graphics window or through the Export node.

The Export to File subnode documentation notes that in batch processes you must pay attention to how filenames synchronize between the batch process file system and the Desktop file system. When in doubt, set explicit output directories and verify the files after the run.

## Batch Licensing

COMSOL's batch license knowledge base states that batch jobs can explicitly request batch licenses by adding `-usebatchlic` to the command. Use this when the licensing setup requires dedicated batch tokens.

## Practical Guidance

- Use Desktop for model authoring and exploratory debugging.
- Use batch for repeatable production solves and sweeps.
- Use methods or Java when the workflow should be regenerated by code.
- Use server/client for multi-process or MATLAB-centric flows.
- Use compile only when the deliverable is an application, not just a solved `.mph`.
